# CLI-Todo
Command Line Task Manager

第一阶段：纯内存 + 命令行
* 学习：模块化、函数设计、单元测试

第二阶段：文件持久化（JSON）
* 学习：序列化、错误处理、边界条件测试

第三阶段：SQLite 数据库
* 学习：SQL基础、数据库抽象层的意义

第四阶段：REST API（用 FastAPI）
* 学习：HTTP、接口设计、集成测试

（可选）第五阶段：加简单前端


我的要求:
* 功能拆分足够小 — 比如"添加任务"这一个功能，我们会先拆成：接收输入、验证输入、存储数据、返回反馈，分步实现。

* 边测试边开发（TDD） — 每个功能先写测试，跑失败，再写实现，跑通过。这是工业界标准的 红-绿-重构 循环。

* 简单实现 + 开拓眼界 — 比如"存储数据"，最简单是 JSON 文件，但我会告诉你工业界为什么用 SQLite、PostgreSQL，或者 ORM 框架。

* 好习惯 — 从一开始就强调：目录结构、命名规范、关注点分离、.gitignore、README。

项目特点:
* 功能边界清晰，容易拆分
* 涉及真实工业项目的核心概念：数据持久化、CRUD、模块化设计
* 可以逐步演进：从内存存储 → 文件存储 → 数据库，每一步都有实际意义
* 后期可以给它加 REST API、前端界面，自然成长为全栈项目
